# VALAR Macro Screener — Macro Pulse Intelligence System

Turns the daily chaos of macroeconomic data into a single, defensible read on the market.

VALAR pulls live indicators from the U.S. Federal Reserve, scores **36 signals across 6 macro
domains** into one **Macro Pulse Score (0–120)**, and classifies the market's regime — from
risk-on to risk-off — on a trading-desk-style dashboard. Every score is traceable back to its
inputs, and an AI-generated *Analyst Note* explains the *why* behind each reading.

🔗 **Live:** https://valar-macro-screener.vercel.app

## Why it's non-trivial

- **A real scoring engine, not a chart wrapper.** A pure, deterministic engine rolls 36
  indicators (scored 1–5 against defined bands) into 6 domain blocks (0–20 each) into one 0–120
  regime score. Every number is auditable back to its inputs and formula.
- **Historical replay.** The engine computes a full snapshot for *any* date, so the app can
  reconstruct history and chart how the regime has shifted over 12 months.
- **Live data + human judgment.** Indicators are pulled and cached from the Federal Reserve
  (FRED) API, with an analyst manual-override path for cases where a raw series is misleading.
- **Fail-safe AI layer.** An institutional "Analyst Note" is generated by Claude Opus 4.8 as a
  *read-only* consumer of the engine — cached by a hash of the underlying scores, schema-validated,
  and non-blocking. If the AI is unavailable, the dashboard renders fully without it.
- **Full admin suite.** Authenticated tooling to run the engine for any date, inspect raw
  traces/formulas, fetch fresh FRED data, and manage manual inputs.

## Stack

Next.js 16 (App Router) · React 19 · TypeScript · PostgreSQL + Drizzle ORM · Tailwind CSS v4 ·
Radix UI · Recharts · TanStack Query · Anthropic SDK (Claude Opus 4.8) · deployed on Vercel.

## How it works

```
FRED API + analyst inputs  ──►  Scoring engine  ──►  Dashboard API  ──►  React dashboard
   (ingestion, cached)          (deterministic,       (snapshots)         (gauge, blocks,
                                 any-date)                                  trend, AI note)
```

Three decoupled layers: **ingestion** feeds a **deterministic scoring engine**, whose snapshots
are served to the **presentation** layer. The AI narrative is a separate, read-only consumer that
can describe a score but never change one.

## Getting started

```bash
npm install
npm run db:push   # sync schema to Postgres
npm run dev       # http://localhost:3000
```

Requires `DATABASE_URL`, `FRED_API_KEY`, and `ADMIN_SESSION_SECRET`. Set `ANTHROPIC_API_KEY` to
enable AI analyst notes (optional — the app degrades gracefully without it).
